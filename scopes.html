AngularJS scope

The scope is the binding part between the HTML (view) and the JavaScript (controller)

The scope is an object with the available properties and methods

The scope is available for both: the view and the controller

When you make a controller in AngularJS, you pass the $scope objest as an argument

Properties, made in the controller, can be reffered to in the view

When adding properties to the $scope object in the controller, the view (HTML) gets axcess to these properties

In the view you don't use the $scope prefix, you just refer to the propertyname, like {{myCar}}.

Understanding the scope

If we consider the AngularJS application consist of:

  • View, which is the HTML;
  • Model, which is the data available for the current view
  • Controller, which is the JavaScript function that makes/changes/removes/controls the data

Then the scope is the Model

The scope is a JavaScript object with properties and methods, which are available for both the view and the controller

Know your scope

Its important to know which scope you are dealing with, at any time

There can be sections in HTML DOM, which can only access certain scopes

When dealing with the ng-repeat directive, each repetition has access to the current repetition object

In this case each repetition object is a string belonging to <li> element and referring by name

Root scope

All applications has a $rootScope which is the scope created on the HTML element that contains ng-app directive

The rootScope available in the entire application.

If a variable has the same name in both the current scope and in the rootScope, the application use the one in current scope

The rootScope's favourite color is

{{color}}

The scope of the controller's favourite color is

{{color}}

The rootScope's favourite color is still

{{color}}
